끈기로 문제를 해결하는 해결사, 개발자 권지현입니다.

프로젝트를 초기 세팅부터 운영까지 웹프론트엔드 전반의 개발을 혼자 해보거나 10명 이상의 팀원과 함께 일하면서 개발자는 단순히 만들어내는 사람이 아니라고 생각하게 되었습니다.

프로젝트를 혼자 개발할 때는 성능 문제도, 기능 개발도 모두 오롯이 책임져야 했기에 사용자의 입장으로 서비스를 자주 테스트하면서 브라우저 성능 개선이 필요하다는 것을 깨닫게 되었습니다. 개발 인원이 충분하지 않기 때문에 클라이언트에서 바로 적용해서 효과를 확인할 방법을 먼저 찾게 되었고 code splitting, 불필요한 라이브러리 교체 등 번들 사이즈를 줄이는 방법을 적용해 브라우저 performance 20점 향상과 FCP, LCP 평균 시간을 약 3초 단축해 브라우저 성능 개선 효과를 경험했습니다.

하이브리드 앱을 위한 기능을 개발할 때는 인앱 브라우저에서 화면이 그려지는 시간이 5초 이상 걸려 UX를 해치는 문제가 있었습니다(특히, 사파리 브라우저에서). 앱 개발자분들과 협업하면서 클라이언트에서 해결할 수 있는 문제를 찾기 위해 노력했습니다. nextjs loading component의 용량이 작을 경우 safari에서 HTML을 로드하지 않고 흰 화면만 노출되는 문제를 찾아 해결하고, react-query의 prefetching과 query-key를 사용해 렌더링 횟수를 줄이고 렌더링 시간을 약 2~3초가량 단축한 경험이 있습니다.

이런 경험이 쌓여 하나의 기능, 하나의 프로덕트를 만들더라도 방법을 끝까지 찾아내고 적용하는 끈기를 가진 개발자가 저, 권지현이라고 자신할 수 있게 되었습니다.
